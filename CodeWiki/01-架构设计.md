# 架构设计

## 整体架构图

```
┌─────────────────────────────────────────────────────────────┐
│                      Electron 主进程                          │
│  (main.js)                                                   │
│  - 窗口生命周期（主窗口、登录窗口）                             │
│  - 系统托盘 & 全局快捷键                                       │
│  - IPC 路由（前端 <-> 主进程）                                 │
│  - Python 解释器自动发现 & 子进程管理                           │
│  - 敏感信息脱敏（日志/IPC）                                     │
└──────────────┬──────────────────────────────┬───────────────┘
               │ preload.js / login-preload.js │
               ▼                               ▼
┌──────────────────────────┐      ┌─────────────────────────────┐
│      渲染进程 (Vue 3)      │      │      Python 桥接进程          │
│  (app.js + index.html)    │      │  (python/*_bridge.py)        │
│  - 页面路由与状态管理       │      │  - 复用 douyin-downloader     │
│  - 组件化 UI               │◄────►│  - stdout JSON Lines 输出     │
│  - IPC API 调用            │      │  - 自动脱敏                   │
└──────────────────────────┘      └─────────────────────────────┘
                                           │
                                           ▼
                                    ┌──────────────┐
                                    │ douyin-downloader
                                    │ (外部项目)    │
                                    │ - API 客户端  │
                                    │ - 下载引擎    │
                                    │ - 数据库/SQLite│
                                    └──────────────┘
```

## 分层说明

### 1. 渲染层（Renderer）
- **技术**：HTML + CSS + Vue 3（无构建工具，直接浏览器运行）。
- **职责**：
  - 提供暗色主题的单页应用界面。
  - 通过 `window.electronAPI` 调用预加载暴露的 IPC 方法。
  - 维护全局 `store` 响应式对象（设置、任务、用户登录态、档案等）。
- **关键文件**：`app/renderer/index.html`、`app/renderer/app.js`、`app/renderer/style.css`。

### 2. 主进程层（Main Process）
- **技术**：Node.js + Electron API。
- **职责**：
  - 创建无边框主窗口（`frame: false`）与独立登录窗口。
  - 实现窗口控制（最小化、最大化、关闭）、托盘图标、全局快捷键。
  - 管理本地设置（`settings.json`）与任务文件（`jobs/*.json`）。
  - 调度 Python 子进程：写入 job 文件 -> `spawn` Python -> 解析 stdout JSON Lines -> 通过 IPC 推送给渲染层。
  - 进程生命周期管理：超时保护、卡死检测（stall timer）、退出清理。
- **关键文件**：`app/main.js`。

### 3. 预加载层（Preload）
- **技术**：Electron `contextBridge`。
- **职责**：
  - 在 `contextIsolation: true` 环境下，为渲染进程暴露安全的、白名单化的 IPC API。
  - 分离主窗口与登录窗口的暴露接口（`login-preload.js` 仅暴露 `completeLogin`）。
- **关键文件**：`app/preload.js`、`app/login-preload.js`。

### 4. Python 桥接层（Bridge）
- **技术**：Python 3 + `asyncio` + `aiohttp`。
- **职责**：
  - 每个业务领域对应一个 `*_bridge.py` 脚本（下载、同步、关系、报表、字幕、云、博主作品、新发布）。
  - 统一通过 `lib.bridge` 读取 job 文件、输出 JSON Lines、自动捕获异常并结束。
  - 通过 `lib.compat.ensure_backend_path()` 将相邻的 `douyin-downloader` 加入 `sys.path`，复用其内核。
- **关键文件**：`python/lib/bridge.py`、`python/*_bridge.py`。

### 5. 核心引擎层（External）
- **说明**：项目本身不包含 `douyin-downloader` 源码，运行时通过路径兼容动态导入。
- **职责**：抖音 HTTP API 封装、Cookie 管理、异步下载、SQLite 存储、报表生成等。
- **期望路径**：`../../douyin-downloader`（相对于 `python/lib/compat.py`）。

## 数据流

### 下载任务示例

1. 用户在渲染层点击下载 -> `window.electronAPI.startDownload(payload)`。
2. `preload.js` 通过 `ipcRenderer.invoke('download:start', payload)` 到达主进程。
3. 主进程 `ipcMain.handle('download:start')`：
   - 生成 `taskId`，将 payload 写入 `jobs/{taskId}.json`。
   - `spawn(python, [download_bridge.py, --job, ..., --task-id, ...])`。
4. `download_bridge.py` 读取 job，调用 `douyin-downloader` 的 `cli_main.download_url()`。
5. Python 进程通过 `BridgeOutput.emit()` 向 stdout 打印 JSON Lines。
6. 主进程 `proc.stdout.on('data')` 解析 JSON，通过 `webContents.send('download:progress', ...)` 推送。
7. 渲染层 `window.electronAPI.onDownloadProgress(callback)` 接收并更新 UI。
8. 任务结束 -> 主进程发送 `download:finished`，清理 job 文件与子进程。

### 同步任务差异

同步（`sync:start`）不经过通用 `runPythonBridge`，而是直接 `spawn` 调用 `douyin-downloader/sync_service.py`，原因是同步任务有独立的超时与卡死保护策略（最大运行 10 分钟，3 分钟无输出视为卡死）。
