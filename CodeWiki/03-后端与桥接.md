# 后端与桥接模块详解

## 1. Python 公共库（python/lib/）

### bridge.py — 统一桥接基类

定义了所有桥接脚本共享的协议与工具：

#### `BridgeContext`
```python
@dataclass
class BridgeContext:
    task_id: str
    task_type: str
```

#### `BridgeOutput`
封装 stdout JSON Lines 输出，**自动脱敏并 flush**。

| 方法 | 说明 |
|------|------|
| `emit(event, data)` | 输出一条 JSON 行，自动注入 `task_id` 与 `task_type`。 |
| `log(message, level)` | 输出脱敏后的日志事件（`{"event":"log","level":"...","message":"..."}`）。 |
| `progress(current, total, message)` | 输出进度事件。 |
| `finished(success, data)` | 输出任务结束事件（幂等，仅第一次有效）。 |
| `error(message)` | 输出错误日志并结束任务（`success=False`）。 |

#### `safe_main(main_func)`
桥接脚本的标准入口装饰器：
1. 解析命令行参数 `--job` 与 `--task-id`。
2. 读取 job JSON 文件。
3. 构造 `BridgeContext` 与 `BridgeOutput`。
4. 调用 `main_func(ctx, job, out)`，异常时自动调用 `out.error()` 并退出码 1。

**使用方式**：
```python
def main(ctx, job, out):
    ...

if __name__ == '__main__':
    safe_main(main)
```

### redactor.py — 敏感信息脱敏

`SensitiveRedactor` 提供统一的脱敏能力，被 `bridge.py` 与主进程 `main.js` 同时使用，确保日志、IPC、导出文件中不会泄露敏感信息。

#### 支持的脱敏规则

| 类型 | 规则示例 | 脱敏效果 |
|------|----------|----------|
| 手机号 | `13800138000` | `138****8000` |
| 敏感 Cookie | `sessionid=abc123` | `sessionid=***` |
| Authorization | `Authorization: Bearer token` | `Authorization: ***` |
| API Key | `AccessKeyId=LTAI...` | `AccessKeyId=***` |
| 字典敏感键 | `{ "api_key": "secret" }` | `{ "api_key": "secr***cret" }` |

#### 关键方法
- `redact_text(text)`：对任意文本脱敏。
- `redact_dict(obj)`：递归对字典中敏感键脱敏，返回新字典。

### compat.py — 兼容层

解决 `douyin-downloader` 作为外部项目的路径导入问题。

| 函数 | 说明 |
|------|------|
| `ensure_backend_path()` | 将 `../../douyin-downloader` 加入 `sys.path`，供本项目 Python 模块复用。 |
| `backend_root()` | 返回 `douyin-downloader` 的绝对路径（若存在）。 |

每个 `*_bridge.py` 开头均调用 `ensure_backend_path()`，随后 `from auth import ...` 等即可正常导入。

## 2. 各桥接脚本职责

### download_bridge.py — 下载桥接

- **职责**：接收 URL 列表与配置，调用 `douyin-downloader` 的 `cli_main.download_url()` 完成下载。
- **关键类**：
  - `DummyDisplay`：屏蔽 CLI 的 Rich 终端输出，防止在 Electron 子进程中写入控制台；同时将进度回调委托给 `ProgressReporter`。
  - `ProgressReporter`：将下载进度转为 JSON Lines 事件（`step`、`item_total`、`item_advanced`、`author`、`title` 等）。
- **错误分类**：`_classify_error()` 将异常转换为用户友好提示（限流、登录失效、404、私密、网络异常等）。
- **输出事件**：`url_start`、`url_result`、`url_error`、`step`、`item_advanced`、`author`、`title`、`log`、`finished`。

---

### relation_bridge.py — 批量关注/取关桥接

- **职责**：根据 `secUids` 列表，调用 `RelationService.iter_batch()` 执行批量关注或取关。
- **参数**：
  - `action`: `"follow"` 或 `"unfollow"`
  - `secUids`: 用户 ID 数组
  - `cookies`: Cookie 字符串
  - `config.minDelay / maxDelay`: 请求间隔（默认 2~4 秒）
  - `config.dryRun`: 是否只模拟不真正调用接口
- **输出事件**：`progress`、`log`、`finished`（含 `summary`）。

---

### report_bridge.py — 报表导出桥接

- **职责**：读取本地 SQLite 数据库，按日期范围生成统计报表，支持按 `date` / `author` / `mode` 聚合，导出 Excel/HTML。
- **关键函数**：
  - `_aggregate(rows, group_by)`：内存聚合明细行。
- **输出**：`report_{timestamp}.xlsx` / `.html`。

---

### transcript_bridge.py — 字幕生成桥接

- **职责**：提取视频音频，调用 OpenAI API 或本地 Whisper 进行语音识别，生成字幕文件。
- **模式**：
  - `api`：调用 `https://api.openai.com/v1/audio/transcriptions`（支持 `gpt-4o-mini-transcribe` 等模型）。
  - `local`：使用 `openai-whisper` 本地模型（`tiny/base/small/medium/large`）。
- **输出格式**：`txt`、`srt`、`vtt`、`json`。
- **额外功能**：中文简繁转换（`opencc`），字幕文件名安全化（过滤特殊字符）。

---

### cloud_bridge.py — 云备份/恢复桥接

- **职责**：将设置文件与数据库加密后上传至云存储，或凭 Token 下载解密恢复。
- **加密**：使用 `cryptography.fernet` 随机生成密钥，AES-128-CBC + HMAC 加密。
- **支持云存储**：
  - **S3 兼容**（通过 `boto3`）：可配置 `endpoint` 以支持 MinIO、Cloudflare R2 等。
  - **阿里云 OSS**（通过 `oss2`）。
- **备份内容**：`settings.json`、`dy_downloader.db`、`cookies.txt`（可选）。
- **Token 机制**：备份完成后返回随机 Fernet Token，用户需妥善保存；恢复时凭同一 Token 下载解密。

---

### user_works_bridge.py — 博主作品列表桥接

- **职责**：根据 `sec_uid` 拉取指定博主的所有作品元数据（不下载视频文件）。
- **分页**：使用 `max_cursor` 翻页，每页 18 条，首页为空时自动重试 2 次。
- **风控策略**：请求间隔 0.8~1.5 秒，配合 `RateLimiter`。
- **输出字段**：`aweme_id`、`title`、`create_time`、`cover`、`share_url`、`author`、`duration`、`digg_count`、`comment_count`。
- **输出事件**：`start`、`progress`、`items`、`done`、`log`、`finished`。

---

### new_releases_bridge.py — 新发布发现桥接

- **职责**：从已同步的关注列表中，找出本地下载过视频的博主，拉取他们的最新作品，过滤掉已下载的 `aweme_id`，返回新作品列表。
- **依赖**：必须开启 SQLite 数据库（用于 `get_downloaded_following_authors` 与 `get_downloaded_aweme_id_set_for_authors`）。
- **流程**：
  1. 查询已下载视频的作者列表（来自 `download_history`）。
  2. 逐个作者拉取作品（每作者上限可配置，默认 30）。
  3. 遇到已下载的 `aweme_id` 即停止翻页（时间倒序优化）。
  4. 汇总所有新作品返回前端。
- **诊断**：若未匹配到作者，输出数据库统计（关注数、aweme 记录数、download_history 成功数等）便于排查。

## 3. 任务文件（Job File）规范

主进程与 Python 桥接通过临时 JSON 文件传递任务参数，位于 `<userData>/jobs/{taskId}.json`。

### 通用字段

```json
{
  "task_type": "download|relation|report|transcript|cloud_backup|cloud_restore|user_works|new_releases",
  "task_id": "uuid"
}
```

### 各任务类型特有字段

| 类型 | 关键字段 |
|------|----------|
| `download` | `urls`, `config`, `cookies` |
| `relation` | `action`, `secUids`, `cookies`, `proxy`, `config` |
| `report` | `dbPath`, `dateFrom`, `dateTo`, `groupBy`, `formats`, `outputDir` |
| `transcript` | `videoPath`, `mode`, `apiKey`, `model`, `formats`, `language` |
| `cloud_backup` | `provider`, `credentials`, `configPath`, `dbPath`, `cookiePath` |
| `cloud_restore` | `provider`, `credentials`, `token`, `outputDir`, `dbPath`, `settingsPath` |
| `user_works` | `sec_uid`, `cookies`, `limit`, `proxy` |
| `new_releases` | `config`, `cookies`, `limits`, `proxy` |

任务完成后，主进程自动删除该 job 文件。
